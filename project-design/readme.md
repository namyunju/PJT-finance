## 기획과 설계

### 요구사항 분석
- 사용자와 이해 관계자의 필요 및 요구를 수집하고 명세화
- 기능적 요구사항
- 비기능적 요구사항
- 핵심 기능 식별 (우선순위 부여)
- 사용자 정의 (페르소나, 사용자 여정)
- 제약 사항 파악 (기술, 일정, 법적 요건)

### 아키텍처와 기술 스택 결정
- 유지 보수성과 확장성 확보
- 팀 역할 분담 명확화
- 스택 선택 기준 : 학습 곡선, 커뮤니티 지원, 성능
- 라이브러리 선정

### 프로젝트 구조와 컨벤션
- 기능별로 폴더 구조화
- 일관된 스타일 유지
- git commit 컨벤션
    - feat : 새로운 기능 추가
    - fix : 버그 수정
    - docs : 문서 수정
    - style : 코드 포멧 수정
    - refactor : 코드 구조 개선

### 프로젝트 세부 설계

**Vue 프로젝트**
- 라우팅 설계
    - 최소한의 Route Tree
    - 중첩 라우팅 vs 단일 라우팅 결정
    - 라우터 가드 전략 세우기
    - RESTful URL 설계

- 컴포넌트 분리 및 책임 정의
    - Views냐 UI냐
    - 하나의 컴포넌트는 하나의 역할만 책임
    - Props와 Emits 설계 (데이터 소유 주체 명확히)

- API 설계 및 연동 구조
    - Back-End API 문서화 (응답 데이터 명세서 작성)
    - AJAX 기술 결정 (공통 인스턴스 설정, 올바른 예외 처리)

- UI / UX 스타일 가이드
    - 디자인 시스템 
    - 전역 CSS vs Scoped CSS
    - 공통 레이아웃 컴포넌트

- Git flow
- 문서화
    - 일정 계획 수립 (WBS & Gantt Chart)
